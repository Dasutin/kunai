# Overview

Kunai is a clean, self-hosted RSS reader. It lets a user add RSS feeds, organize them into folders, browse unread or saved articles, search content, tag items, fetch readable article bodies, import/export OPML, and install the web app as a PWA.

## Main Capabilities

- Add, edit, delete, mute, refresh, move, and reorder feeds.
- Organize feeds into folders and reorder folders.
- Browse items in list, card, and magazine layouts.
- Filter by unread state, selected feed, selected folder, tags, saved items, and search text.
- Mark individual items or whole scopes as read.
- Save and unsave articles.
- Manage tags and merge tags.
- Fetch readable article content through Mozilla Readability when enabled.
- Import and export OPML.
- Configure user-facing defaults such as theme, refresh interval, retention, and default layout.
- Use companion browser extensions to show unread count and open the Kunai instance.

## Current Maturity

Kunai is functional and buildable, but it should be treated as a trusted-network personal app in its current form. The inspected code has no authentication layer, uses permissive CORS, has limited runtime validation on several endpoints, and does not yet include an automated test suite. See [Codebase Review](codebase-review.md) and [Recommendations](recommendations.md) for the review backlog.

## Repository Layout

```text
client/              Vite React web app
client/public/       PWA manifest, service worker, and icons
server/              Express API, RSS scheduler, SQLite repositories
server/migrations/   Versioned SQLite schema migrations
shared/              TypeScript API/domain contracts shared by client and server
extensions/chrome/   Chrome Manifest V3 companion extension
extensions/firefox/  Firefox companion extension
mobile/ios/          SwiftUI starter companion app
dist/                Build output, generated by npm run build
data/                Local SQLite data directory, generated at runtime
```

## Technology Stack

- Node.js 20 or newer.
- TypeScript with strict mode.
- Express for the API and static production hosting.
- better-sqlite3 for local persistence.
- rss-parser for feed parsing.
- sanitize-html for HTML cleanup.
- jsdom and `@mozilla/readability` for readable article extraction.
- React 18 and Vite for the web app.
- Docker for production packaging.

## Runtime Shape

In development, the API and client run as separate processes. The client is served by Vite on port `5173` and proxies API requests to Express on port `3000`.

In production, the React client is built to `dist/client`, the TypeScript server is built to `dist/server`, and Express serves both `/api/*` routes and the static client from a single process.

The server refreshes feeds at startup and through a recurring scheduler. The web client also triggers refreshes while visible on its own interval, so refresh behavior currently comes from both server-side and client-side loops.
